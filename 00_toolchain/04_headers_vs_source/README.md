# Topic 02 — Headers vs Source: Strategic Decisions

# Topic
Understanding what goes where in a C++ project, and why these decisions matter for compile times and dependencies in large codebases [2].

---

# Key Concepts

# The Hidden Cost of `#include`
Every `#include` literally pastes header contents into your translation unit. Heavy headers like `<vector>` add tens of thousands of lines. This compounds across hundreds of files in production codebases.

# Forward Declarations
Tell the compiler "this type exists" without giving full details:

```cpp
class Point;  // Forward declaration

class Shape {
public:
    void setCenter(const Point& p);  // OK — only needs forward decl
    const Point* getCenter() const;  // OK — pointer
private:
    Point* center_;                  // OK — pointer
    // Point center_;                // ERROR — needs full type for size
};

> Key Takeaways
- Headers should be MINIMAL — include only what's truly needed for the interface
- Source files can include heavy headers freely — only one translation unit affected
- Forward declarations reduce compile-time dependencies dramatically
- In large codebases, header hygiene directly impacts build times
- The pattern of "forward declare in header, full include in source" is industry standard

> The Three File Categories in C++ Builds
┌─────────────────────────────────────────────────┐
│ SOURCE FILES (.cpp)                             │
│   → Pass to compiler                            │
│   → Become .obj files                           │
│   → Names in your build command                 │
└─────────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│ HEADER FILES (.hpp / .h)                        │
│   → Never pass to compiler directly             │
│   → Pulled in by #include                       │
│   → NEVER name them in build command            │
└─────────────────────────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│ OBJECT FILES (.obj)                             │
│   → Generated by compiler from .cpp             │
│   → Linker combines them                        │
│   → Usually managed automatically by `cl`       │
└─────────────────────────────────────────────────┘
